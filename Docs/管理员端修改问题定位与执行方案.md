# 管理员端修改：问题定位与详细执行方案

> 项目：复旦大学附属儿科医院技能培训中心前端 Demo  
> 适用范围：`03（教务）管理员端`  
> 文档目的：供执行能力较弱的 AI 按步骤修改代码  
> 当前阶段：问题定位与执行规划，不包含具体业务后端实现

## 一、执行前必读

本项目是纯前端 Demo，使用 Vue 3、Arco Design 和本地 Mock Data，不接后端。

管理员端入口文件为：

```text
03（教务）管理员端/index.html
```

实际页面由以下公共文件动态加载：

```text
shared/nav-data.js
shared/nav-render.js
shared/module-loader.js
shared/modules/*
```

执行修改时必须遵守以下规则：

1. 先运行 `git status --short`，确认当前工作区状态。
2. 先保存或查看当前 `git diff`。
3. 当前相关文件已有未提交修改，禁止直接覆盖整份文件。
4. 只允许做小范围补丁修改。
5. 禁止修改 `03（教务）管理员端/index.single.editable.html`。
6. 不要全局修改字体变量，否则会影响教师端、学生端和其他管理员端。
7. 每完成一个模块，立即做语法检查和页面回归。
8. 不要一次修改全部模块后才测试。
9. 不要新增后端接口或复杂依赖。
10. 不要使用 Emoji 作为图标。

## 二、问题定位结论

### 2.1 课程开发：管理员端可能出现空白

涉及文件：

```text
shared/modules/course-dev/course-dev.js
shared/modules/course-dev/course-dev.css
shared/module-loader.js
shared/nav-data.js
```

`shared/module-loader.js` 已经配置“课程开发”模块：

```js
'课程开发': {
  css: ['modules/course-dev/course-dev.css'],
  js: ['modules/course-dev/course-dev.js']
}
```

管理员导航也存在“课程开发”入口。

但是 `course-dev.js` 中的 `shouldHandle()` 只允许教师角色：

```js
function shouldHandle() {
  return document.body.dataset.role === 'teacher'
    && document.body.dataset.active === '课程开发';
}
```

结果是：

1. 管理员点击“课程开发”。
2. 模块加载器成功加载 JS 和 CSS。
3. `course-dev.js` 判断当前角色不是 `teacher`。
4. 模块不渲染。
5. 页面内容区可能为空白。

这是当前必须最先修复的入口级问题。

### 2.2 课程开发：所谓“解析”目前只是模拟

当前创建课程页面写有：

```text
支持 Word、PDF、PPT 等格式，上传后自动解析课程内容。
```

但是 `simulateUpload()` 只执行：

```js
this.uploaded = true;
this.$message.success('已读取《儿童导尿术教案.docx》');
```

当前不存在：

- 文件解析过程；
- 解析进度；
- 解析成功或失败状态；
- 解析字段结果；
- 待用户确认字段；
- 解析结果到编辑页的数据传递。

因此当前页面不能宣称已经完成真实解析，只能实现前端 Demo 级的模拟解析流程。

### 2.3 课程开发：示范库存在重复数据

当前存在两套“课程开发示范库”：

1. `course-dev.js` 中的课程创建示范库弹窗；
2. `resource-library.js` 中的课程开发示范库。

两处分别维护 Mock Data，后续修改时容易出现：

- 课程名称不一致；
- 分类不一致；
- 教案、视频、模板数量不一致；
- 管理员维护后教师端没有同步。

需要抽取公共数据源。

### 2.4 课程开发：AI 已位于编辑页

课程编辑页已经存在“课程开发助手”：

- 编辑页右下角有 AI 入口；
- 可以打开 AI 助手弹层；
- 已有课程概述、风险提醒和快捷操作；
- 已有输入框。

所以“AI 支持在编辑页面”这一位置要求已经满足。

但具体功能尚未确定，本轮不能继续扩展功能，也不能自作主张对接真实 AI。

### 2.5 课程开发：内部导航字号偏小

以下位置大多使用 12px：

```text
.cd-main-tabs button
.cd-anchor-list a
.cd-overview-nav a
```

可按照“字号大一号”从 12px 调整为 13px。

### 2.6 管理员工作台：背景、文案、图标和字号

涉及文件：

```text
shared/modules/dashboard/dashboard.js
shared/modules/dashboard/dashboard.css
```

当前问题：

1. 页面背景硬编码为 `#f4f5f8`。
2. 大量服务说明文案是统一拼接的：

   ```text
   进入某模块，办理或查看“某业务”相关业务。
   ```

3. 这些说明没有提供额外信息，属于无意义重复文案。
4. 服务标题主要为 14px。
5. 分类页签和分组标题主要为 13px。
6. 图标已使用 Lucide 和彩色浅底容器，但不是严格意义的实心图标。

### 2.7 左侧导航字号偏小

涉及文件：

```text
shared/nav.css
```

当前主菜单和子菜单使用：

```css
font-size: var(--font-md);
```

`--font-md` 当前为 13px。

建议改为局部 14px，并增加中等字重。

### 2.8 班牌和大屏：当前操作与需求不一致

涉及文件：

```text
shared/modules/brand-display-management/brand-display-management.js
shared/modules/brand-display-management/brand-display-management.css
```

当前顶部包含：

- 班牌管理；
- 信息大屏；
- 通知公告。

当前设备列表每行包含四个操作：

- 详情；
- 编辑；
- 停用/启用；
- 删除。

需求希望改为三个操作：

- 编辑；
- 删除；
- 通知。

“顶部去通知”结合当前代码结构，建议理解为：

> 删除顶部“通知公告”页签，但保留针对单个班牌或大屏发送通知的能力。

### 2.9 空间资产：领导数据大屏已有空间区域

涉及文件：

```text

```

当前领导数据大屏已经存在：

- 教学空间列表；

1. 管理员左侧仍有“空间资产管理”入口。
2. 数据大屏和空间资产管理维护了不同的空间 Mock Data。
3. 完整的资产增删改页面不适合直接嵌入领导大屏。

### 2.11 教学资源库：弹题目前放错业务位置

涉及文件：

```text
shared/modules/resource-library/resource-library.js
shared/modules/resource-library/resource-library.css

```

教学资源库中已经实现：

- 单选题、多选题、判断题；

从业务流程看，更合理的位置是：

```text
课程开发
→ 编辑课程
→ 添加资源
→ 选择视频
→ 设置弹题
```

但需求明确说明要到评估功能模块时再讨论，因此本轮不能迁移或删除。

## 三、详细执行方案

## 3.1 阶段零：保护当前工作区

执行以下命令：

```powershell
git status --short
git diff -- shared/modules/course-dev/course-dev.js
git diff -- shared/modules/dashboard/dashboard.js
git diff -- shared/nav-data.js
git diff -- shared/nav.css
```

要求：

1. 不清理用户已有修改。
2. 不运行 `git reset --hard`。
3. 不运行 `git checkout -- 文件名`。
4. 不重建整个模块文件。
5. 只修改与本需求直接相关的代码。

## 3.2 阶段一：修复管理员课程开发空白

修改文件：

```text
shared/modules/course-dev/course-dev.js
```

找到：

```js
function shouldHandle() {
  return document.body.dataset.role === 'teacher'
    && document.body.dataset.active === '课程开发';
}
```

改为：

```js
function shouldHandle() {
  return ['teacher', 'admin'].includes(document.body.dataset.role)
    && document.body.dataset.active === '课程开发';
}
```

不要加入以下角色：

- student；
- scheduler；
- backend；
- material。

验收：

1. 管理员进入“课程开发”不再空白。
2. 教师进入“课程开发”仍正常。
3. 学生端不出现课程开发编辑器。
4. 离开课程开发页面后 Vue 实例可以正常卸载。
5. 控制台没有重复挂载警告。

## 3.3 阶段二：补全前端 Demo 解析流程

如果“基础资料库（解析的问题）”指上传教案解析，则执行本阶段。

如果“基础资料库”是一个独立栏目名称，暂停本阶段，先确认栏目定义。

### 需要增加的状态

在课程开发 Vue Data 中增加：

```js
parseStatus: 'idle',
parseProgress: 0,
parseError: '',
parsedCourse: null
```

状态含义：

```text
idle       未选择文件
reading    正在读取
parsing    正在解析
success    解析完成
error      解析失败
```

### 模拟解析流程

上传后按顺序展示：

1. 已选择文件；
2. 正在读取文件；
3. 正在识别课程结构；
4. 正在整理教学目标和章节；
5. 解析完成。

解析完成后展示：

```text
课程名称
课程类型
授课对象
课程负责人
教学目标
课程章节数
识别可信度
待确认字段
```

### Mock 解析结果示例

```js
parsedCourse: {
  name: '儿童导尿术（男性）',
  type: '临床技术性技能课程',
  audience: '住培一年级',
  owner: '刘国强',
  objectives: [
    '掌握儿童导尿术适应证与禁忌证',
    '完成标准无菌导尿操作',
    '识别并处理常见风险'
  ],
  sectionCount: 6,
  confidence: 92,
  pendingFields: ['课程学时', '考核方式']
}
```

### 进入编辑页

`enterEditor()` 必须优先使用 `parsedCourse`：

```js
this.editingCourse = {
  name: this.parsedCourse.name,
  type: this.parsedCourse.type,
  audience: this.parsedCourse.audience,
  owner: this.parsedCourse.owner,
  teacherTeam: ['刘国强', '王雨桐', '蔡小芳'],
  cover: coverUrl
};
```

禁止继续无条件写死所有课程数据。

### 解析失败

必须提供：

- 失败提示；
- 重新解析按钮；
- 返回重新选择文件；
- 不允许在失败状态直接进入编辑页。

### Demo 文案要求

因为没有后端，不要写“系统已真实读取 Word 内容”。

推荐文案：

```text
已完成演示解析，识别结果请在进入编辑页前确认。
```

要求：

- 不增加背景图片；

### 删除无意义文案

删除管理员服务卡中的：

```html
<small>{{ svc.desc }}</small>
```

原搜索逻辑：

```js
s.label.toLowerCase().indexOf(keyword) !== -1
|| s.desc.toLowerCase().indexOf(keyword) !== -1
```

```js
s.label.toLowerCase().includes(keyword)
|| s.category.toLowerCase().includes(keyword)
```

不要删除真正有业务价值的内容，例如：

- 服务名称；
- 分类名称；
- 搜索框；
- 常用服务；
- 全部服务。

- Lucide 线性图标；
- 图标外有浅色圆角底块；
- 不同业务使用不同语义色；
- 图标容器大小统一；
- 不使用 Emoji。

不要直接给 Lucide SVG 设置：

```css
fill: currentColor;
```

这可能导致图标形态异常。

如果客户明确要求真正实心图标，需要另行确认是否允许偏离 Lucide 设计规范。

### 字号

```css

  font-size: 15px;
}

  font-size: 14px;
}

.admin-group-title {
  font-size: 14px;
}
```

页面标题保持 18px。

验收：

1. 无重复说明文案。
2. 服务名称仍清晰。
3. 搜索服务名称有效。
4. 搜索分类名称有效。
5. 图标大小和背景统一。
6. 教师、学生工作台无变化。
