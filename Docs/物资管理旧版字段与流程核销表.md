## 4. 页面信息架构约束

每个物资一级页面统一采用以下结构：

1. `app-page-header`：页面标题、数量/状态摘要、一级动作。
2. `app-view-tabs`：同一业务对象的视图切换；没有真实并列视图时不得硬加。
3. `app-control-bar`：搜索、筛选、排序、重置。
4. `app-context-bar`：风险、冲突、流程说明等临时上下文。
5. `app-selection-bar`：仅在勾选数据后出现批量动作。
6. `app-page-body`：表格、卡片或详情正文。

列表首屏只展示识别和决策所需字段；其余字段进入详情或展开区。隐藏不是删除，所有本表字段必须仍可查看和编辑。

## 5. 最终验收口径

- 六个入口逐个打开，不出现空白页、脚本报错或错误标题。
- 搜索、筛选、标签页和批量选择均改变真实数据状态，不做静态摆设。
- 任取一个设备档案和一个耗材档案，能找到本表中各自全部字段。
- 借用—归还、故障—维修完成、创建盘点—盘点完成至少各跑通一次。
- 1280×720、1440×900、1920×1080 下无横向溢出，标题、筛选和主要动作不被裁切。
